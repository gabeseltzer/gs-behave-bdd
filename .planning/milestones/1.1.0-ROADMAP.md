# Milestone 1.1.0: Config File Watching

**Status:** SHIPPED 2026-04-17
**Phases:** 4-6 (3 phases)
**Total Plans:** 9

## Overview

1.1.0 made the auto-discovery behavior from 1.0.0 reactive: the extension now watches all 5 behave config formats, silently rebuilds the test tree on config edits, and guards test execution when a workspace's config is malformed. End-to-end coverage was added as the 14th integration suite, locked in behind a 3-run flakiness gate.

## Milestone Summary

**Decimal Phases:** None — Phases 4, 5, 6 all sequential.

**Key Decisions:**

- Config watcher callback routes through `configurationChangedHandler(undefined, undefined, true)` — preserves integration test guard, log clearing, watcher rebuild, and `clearNotifiedErrors=true`
- Brace-expansion glob `{behave.ini,.behaverc,setup.cfg,tox.ini,pyproject.toml}` — bare filenames silently fail (VS Code bug #164925)
- 500ms debounce mandatory — stale file read race on `onDidChange` (VS Code bug #72831)
- Run guard reads discovery cache via `getDiscoveryEntry()`, NOT from `WorkspaceSettings` snapshot (staleness-safe)
- Non-blocking guard UX: user can always proceed ("Run Anyway"); hard blocking was explicitly out of scope
- Dedicated test fixtures (D-05): suites that mutate fs state get their own `example-projects/` directory
- `waitForTestTree` predicate polling (D-08) replaces wall-clock sleeps across integration suites
- Phase 5 automation obsoleted 5 Phase 4 human UAT items; audit `gaps_found` status was cleared by Phase 5 + Phase 6 execution

**Issues Resolved:**

- 1.1.0-MILESTONE-AUDIT `gaps_found` → resolved: TEST-08 closed (Phase 5), tech-debt + admin hygiene closed (Phase 6)
- Windows FileSystemWatcher delete latency (1-5s) — handled in Phase 5 fix `b54de65` via ≥15000ms timeouts
- Pitfall 14 confirmed: `configWatcher` bypasses `configurationChangedHandler` so `integrationTestRun = true` does not prevent watcher from firing in tests

**Issues Deferred:**

- Subdirectory config scanning (DISC-07, depth-3) → 1.2.0 candidate
- Multiple feature paths via `featuresUris[]` (DISC-08) → 1.2.0 candidate
- Multiple behave projects per workspace (MULTI-01) + `Behave BDD: Select Project` quick-pick (MULTI-02) → Milestone 3

**Technical Debt Incurred:**

- None — Phase 6 cleared all outstanding code-review findings and admin hygiene before milestone close
