# Quick Task 260515-j3n: CodeLens references should update on feature file edits

## Problem

`StepCodeLensProvider` shows "N references" above each step definition in Python files.
The count goes stale when a `.feature` file is edited: step mappings are rebuilt
(`fileParser.reparseFile` → `rebuildStepMappings`, fileParser.ts:642), but:

1. The feature-file reparse path does not fire `parser.onStepMappingsRebuilt` (only the Python debounce path does, fileParser.ts:750).
2. `StepCodeLensProvider` has no `onDidChangeCodeLenses` event emitter, so VS Code only re-queries it on natural triggers (`.py` document change).

Result: editing a `.feature` file does not refresh the CodeLens count in any open `.py` file.

## Tasks

### Task 1: Fire `onStepMappingsRebuilt` from feature reparse path

**File:** `src/parsers/fileParser.ts`

After the `rebuildStepMappings(root, wkspSettings.featuresUri)` call in the feature reparse branch (around line 642), invoke `this.onStepMappingsRebuilt?.(wkspSettings.featuresUri)` so subscribers know step mappings changed.

**Verify:** Existing tests in `test/unit/parsers/stepDefDiscoveryOrdering.test.ts` and `reparseFileDebounce.test.ts` still pass. Add a new test asserting the callback fires after a feature-file reparse.

### Task 2: Add `onDidChangeCodeLenses` to `StepCodeLensProvider`

**File:** `src/handlers/codeLensProvider.ts`

Add a `vscode.EventEmitter<void>` field, expose it as the `onDidChangeCodeLenses` event (the VS Code `CodeLensProvider` optional event), and add a public `refresh()` method that fires it.

**Verify:** `test/unit/handlers/codeLensProvider.test.ts` still passes; add a test confirming `refresh()` fires the event.

### Task 3: Wire CodeLens refresh into the parser hook

**File:** `src/extension.ts`

Hold a reference to the `StepCodeLensProvider` instance (around line 462) and call `codeLensProvider.refresh()` inside the existing `parser.onStepMappingsRebuilt` handler (around line 388).

**Verify:** ESLint clean; `npm run test:unit` passes.

## Definition of Done

- `npx eslint src --ext ts` exits 0
- `npm run test:unit` passes
- New unit tests cover: (a) feature reparse fires `onStepMappingsRebuilt`, (b) `StepCodeLensProvider.refresh()` fires `onDidChangeCodeLenses`.
