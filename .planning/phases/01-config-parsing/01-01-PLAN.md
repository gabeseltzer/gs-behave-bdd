**searchConfigFiles()** (per DISC-01, DISC-05):
Iterate CONFIG_FILES in order. For each: build URI via `vscode.Uri.joinPath(wkspUri, filename)`, check `fs.existsSync(fileUri.fsPath)`. If exists, dispatch to `parseIniConfig(fileUri)` or `parseTomlConfig(fileUri)` based on `format`. If the parser returns a non-undefined result, return it immediately (first match wins = priority order). If no file yields a result, return `undefined`.

**parseIniConfig(fileUri)** (per DISC-02, DISC-06):
Hand-rolled INI parser matching Python configparser continuation-line semantics:
1. Read file via `fs.readFileSync(fileUri.fsPath, 'utf8')` wrapped in try/catch (return `undefined` on read failure).
2. Split content on `/\r?\n/`.
3. Track state: `inBehaveSection` (boolean), `collectingPaths` (boolean), `pathsLines` (string[]).
4. For each line:
   - If `trimmed.startsWith('[')`: check if `trimmed === '[behave]'` to enter section, or if already in section then `break` (left the section).
   - If not in `[behave]` section: `continue`.
   - If blank line (`trimmed === ''`) or comment (`trimmed.startsWith('#')` or `trimmed.startsWith(';')`): set `collectingPaths = false` and `continue`. This is critical -- blank/comment lines break continuation per Python configparser.
   - If line matches `/^paths\s*=/`: extract value after `=`, set `collectingPaths = true`, start new `pathsLines` array.
   - If `collectingPaths` and raw line starts with whitespace (`/^\s/.test(line)`): push `trimmed` to `pathsLines` if non-empty (continuation line).
   - Otherwise: set `collectingPaths = false` (different key encountered).
5. After loop: if `!inBehaveSection` return `undefined` (DISC-06). If `pathsLines.length === 0` return `undefined`.
6. Build rawPaths by mapping and filtering empty strings. If empty after filter, return `undefined`.
7. Return `buildResult(fileUri, 'ini', rawPaths)`.

**parseTomlConfig(fileUri)** (per DISC-03, DISC-06):
1. Read file via `fs.readFileSync(fileUri.fsPath, 'utf8')` wrapped in try/catch (return `undefined` on read failure).
2. Parse via `parseToml(content)` wrapped in try/catch (return `undefined` on malformed TOML -- Phase 3 adds UX warning).
3. Cast result: `const tool = (parsed as Record<string, unknown>)?.tool as Record<string, unknown> | undefined`.
4. Access: `const behave = tool?.behave as Record<string, unknown> | undefined`. If `!behave` return `undefined` (DISC-06).
5. Access: `const paths = behave.paths`. Check `Array.isArray(paths)`. If not array or empty, return `undefined` (handles TOML Pitfall 2: scalar string instead of array).
6. Build rawPaths: `paths.map(String).filter(p => p.length > 0)`. If empty, return `undefined`.
7. Return `buildResult(fileUri, 'toml', rawPaths)`.

**resolvePaths(rawPaths, configFileUri)** (per DISC-04, D-03):
1. Compute config directory: `const configDirUri = vscode.Uri.joinPath(configFileUri, '..')`.
2. Take only `rawPaths[0]` (v1 single-path per D-03, D-04).
3. Check if absolute path: `rawPath.startsWith('/') || /^[a-zA-Z]:[\\/]/.test(rawPath)`. If absolute, return `vscode.Uri.file(rawPath)`.
4. If relative, return `vscode.Uri.joinPath(configDirUri, rawPath)`.

**buildResult()** helper:
```typescript
function buildResult(
  configFileUri: vscode.Uri,
  format: 'ini' | 'toml',
  rawPaths: string[]
): BehaveConfigResult {
  return {
    configFileUri,
    format,
    rawPaths,
    resolvedPath: resolvePaths(rawPaths, configFileUri),
  };
}
```

- Use `vscode.Uri.joinPath()` for ALL path construction in `src/` -- never `path.join()`.
- Return `undefined` for all error cases -- never throw from internal helpers.
- Do NOT add logging (Phase 3 adds UX).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Filesystem -> Parser | Config file content is read from disk; may contain arbitrary text |
| smol-toml parse | Third-party library processes untrusted TOML content |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | T (Tampering) | configParser.ts fs.readFileSync | accept | Config files are in the user's own workspace; tampering requires local filesystem access which is already trusted by VS Code |
| T-01-02 | D (Denial of Service) | parseIniConfig line-by-line loop | mitigate | Parser operates on bounded input (config files are typically <100 lines); readFileSync reads entire file but config files are negligible size. No unbounded loops -- loop terminates at end of lines array. |
| T-01-03 | D (Denial of Service) | parseToml via smol-toml | mitigate | smol-toml parse is wrapped in try/catch; malformed TOML returns undefined instead of crashing the extension. No recursive or exponential parsing paths in TOML 1.0 spec. |
| T-01-04 | I (Information Disclosure) | configParser.ts error handling | accept | Parser returns undefined on errors; no error messages are logged or shown to user in Phase 1. No sensitive data in config files (paths only). Phase 3 will add user-facing error messages. |
| T-01-05 | S (Spoofing) | CONFIG_FILES priority order | accept | Priority order is hardcoded matching behave's own order; a user who controls the workspace can place any config file they want, but this is expected behavior (their own workspace). |
</threat_model>
