Create a new test fixture at `example-projects/migration-stale/` that pre-seeds
all three Phase 15 + Phase 16 legacy settings keys at WorkspaceFolder scope.
The fixture is the load-bearing artifact for Phase 17's combined-coverage
integration test (per CONTEXT.md D-05, D-06, D-07, D-08).

Purpose: One Dev Host launch against this fixture must exercise — through the
production activation loop — Phase 15's `suppressMultiConfigNotification` →
`suppressedNotifications` migration AND Phase 16's two-namespace `featuresPath`
→ `featuresPaths` migration AND the post-migration notification path. Per
CONTEXT.md §Specifics, the fixture seeds both `gs-behave-bdd.featuresPath` and
`behave-vsc.featuresPath` (different values) plus
`gs-behave-bdd.suppressMultiConfigNotification: true`.

Output: ~6 small files under `example-projects/migration-stale/`, all
pre-committed (D-08), with a sibling `settings.template.json` that Plan 02 uses
to restore the baseline after each suite run.
</objective>
