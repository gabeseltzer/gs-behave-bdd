<objective>
Source-tree singular `featuresPath` cleanup: remove the package.json schema entry
(DEP-01), collapse `src/settings.ts`'s 4-rung precedence ladder to 3 rungs (D-15 / DEP-05),
simplify `src/common.ts`'s `hasFeaturesFolder()` discovery branch to use plural-only
(D-16 / DEP-05), and drop the singular surface from the `TestWorkspaceConfig` mock in
`src/testWorkspaceConfig.ts` (D-17 / DEP-06).

Purpose: by the end of this plan, no `src/` production-code path reads the singular
`featuresPath` setting. The migration helper from Plan 03 / Plan 04 has already moved
every legacy value into `featuresPaths` at the appropriate scope at activation, so the
collapsed ladder operates on already-migrated data. The schema removal is the final
contract change: VS Code stops surfacing `featuresPath` in the settings UI.

Output: 4 modified files. Test fixtures that consumed `TestWorkspaceConfig`'s singular
surface (e.g., `BASE_CFG.featuresPath`) will break — Plan 06 handles that cascade.

Critical ordering note: this plan runs in Wave 4, AFTER Plan 04's activation-loop
wiring lands. That means at the time this code ships, every existing user with a
legacy `featuresPath` in settings.json will have it auto-migrated on the next activation
to `featuresPaths`. The collapsed ladder is therefore safe — it never has to handle a
"user with legacy singular only" case at runtime.

Constraints honored:
- **Strict-undefined throw pattern preserved** for `featuresPathsCfg` read (the comment
  at `settings.ts:189` says "D-12: no throw on undefined" — the plural is registered
  with default `[]` so `get()` always returns at least the default; the `undefined`-tolerance
  is defensive for VS Code edge cases).
- **Backward compatibility:** users with explicit `projectPath` keep zero behavior change
  (CLAUDE.md project constraint). Users with explicit `featuresPath` get auto-migrated
  by Plan 03/04 — they see the migration notification once and their settings shift to
  `featuresPaths` with no behavior change to test discovery.
- **Bundle size unchanged:** no new dependencies; this plan only deletes code.
</objective>
