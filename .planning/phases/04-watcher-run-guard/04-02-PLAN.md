<objective>
Implement a run guard that intercepts test execution when any workspace with queued tests has a malformed config. The guard reads from the discovery cache via `getDiscoveryEntry()`, shows a warning dialog with three action buttons, and either proceeds, opens the config file, or cancels based on user choice. The guard fires for both regular runs and debug sessions. Unit tests verify all response paths and workspace scoping.

Purpose: Users are warned before running tests against a workspace with a malformed config, preventing confusing test failures.
Output: Modified `src/runners/testRunHandler.ts`, extended `test/unit/runners/testRunHandler.test.ts`
</objective>

```typescript
export async function checkRunGuard(
  request: vscode.TestRunRequest,
  ctrl: vscode.TestController
): Promise<boolean> {

  // GUARD-04: Collect workspace URIs only for tests that are actually queued
  const wkspUriSet = new Set<string>();
  const items = request.include ?? convertToTestItemArray(ctrl.items);
  for (const item of items) {
    if (request.exclude?.includes(item)) continue;
    const wkspSettings = getWorkspaceSettingsForFile(item.uri);
    if (wkspSettings) {
      wkspUriSet.add(uriId(wkspSettings.uri));
    }
  }

  // GUARD-01: Check discovery cache for configError in each queued workspace
  const brokenWorkspaces: { wkspUri: vscode.Uri; filename: string; configFileUri: vscode.Uri }[] = [];
  for (const wkspUri of getUrisOfWkspFoldersWithFeatures()) {
    if (!wkspUriSet.has(uriId(wkspUri))) continue;
    const entry = getDiscoveryEntry(wkspUri);
    if (entry?.configError) {
      brokenWorkspaces.push({
        wkspUri,
        filename: basename(entry.configError.configFileUri),
        configFileUri: entry.configError.configFileUri,
      });
    }
  }

  if (brokenWorkspaces.length === 0) return true; // no issues â€” proceed

  // GUARD-02: Show warning with three options (D-04 message format, D-06 multi-root behavior)
  const filenames = brokenWorkspaces.map(b => `'${b.filename}'`).join(', ');
  const msg = `Config file ${filenames} has parse errors. Tests may not discover correctly.`;

  // D-14: Audit trail in output channel
  for (const bw of brokenWorkspaces) {
    config.logger.logInfo(`Run guard: config error in '${bw.filename}' â€” user prompted`, bw.wkspUri);
  }

  const choice = await vscode.window.showWarningMessage(
    msg,
    'Run Anyway',
    'Open Config File',
    'Cancel'
  );

  if (choice === 'Run Anyway') {
    return true; // proceed with test execution
  }

  if (choice === 'Open Config File') {
    // D-06: Open the first broken config file and cancel the run
    await vscode.commands.executeCommand('vscode.open', brokenWorkspaces[0].configFileUri);
    return false;
  }

  // 'Cancel' or dismiss (undefined) â€” cancel the run
  return false;
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Discovery cache -> Run guard | Guard reads configError from in-memory cache populated by config file parsing |
| Run guard -> User dialog | VS Code showWarningMessage presents choices; user response gates test execution |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04 | Spoofing | getDiscoveryEntry | accept | Cache is internal to extension process; no external input. The configError is populated by the extension's own parser. No spoofing vector. |
| T-04-05 | Denial of Service | checkRunGuard | accept | Guard is a single async call that checks an in-memory Map and shows at most one dialog. No amplification risk. "Run Anyway" ensures the user can always proceed. |
| T-04-06 | Elevation of Privilege | vscode.commands.executeCommand('vscode.open') | accept | Opens user's own config file in the editor. No privilege escalation â€” VS Code's own `vscode.open` command. |
</threat_model>
