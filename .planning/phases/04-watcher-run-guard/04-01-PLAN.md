<objective>
Implement config file watching that silently re-discovers features when any of the 5 behave config files (behave.ini, .behaverc, setup.cfg, tox.ini, pyproject.toml) change at a workspace root. The watcher fires on create, change, and delete events with 500ms per-workspace debounce, then refreshes the discovery cache, updates the UX (output channel, diagnostics, notifications), and triggers a test tree rebuild. Unit tests verify debounce behavior, timer independence, and lifecycle management.

Purpose: Users see the test tree update automatically when a behave config file changes, without any manual action.
Output: `src/watchers/configWatcher.ts`, modified `src/extension.ts`, `test/unit/watchers/configWatcher.test.ts`
</objective>

```typescript
  for (const timer of configDebounceTimers.values()) {
```

**Function signature** (per D-08, mirrors workspaceWatcher.ts):
```typescript
export function startWatchingConfigFiles(
  wkspUri: vscode.Uri,
  ctrl: vscode.TestController,
  testData: TestData,
  parser: FileParser,
  onConfigChanged: (wkspUris: vscode.Uri[], clearNotifiedErrors: boolean) => void
```

The `onConfigChanged` callback parameter receives `updateDiscoveryUX` from extension.ts. This avoids exporting `updateDiscoveryUX` and avoids circular imports (per RESEARCH.md Assumption A1 — callback approach).

**Implementation:**
1. Create a `vscode.RelativePattern(wkspUri, CONFIG_GLOB)` -- MUST use brace-expansion glob, NOT bare filenames (VS Code bug #164925, PITFALL-02).
2. Create watcher via `vscode.workspace.createFileSystemWatcher(pattern)`.
3. Create a unified handler function that all three events call (per D-03 — no special-casing for delete):
   ```typescript
   const handler = (eventUri: vscode.Uri, eventType: string) => {
     if (eventUri.scheme !== 'file') return;
     diagLog(`configWatcher: ${eventType} detected for ${eventUri.fsPath}`);

     const key = wkspUri.path;
     const existing = configDebounceTimers.get(key);
     if (existing) {
       clearTimeout(existing);
       diagLog(`configWatcher: debounce timer reset for ${wkspUri.path}`);
     }

     const timer = setTimeout(async () => {
       configDebounceTimers.delete(key);
       try {
         const filename = eventUri.path.split('/').pop() ?? 'config file';
         config.logger.logInfo(
           `Config file changed: ${filename} — re-discovering features...`,
           wkspUri
         );
         // Direct cache invalidation — do NOT call configurationChangedHandler (PITFALL-04)
         getUrisOfWkspFoldersWithFeatures(true);
         onConfigChanged([wkspUri], true);  // clearNotifiedErrors=true per WATCH-06
         parser.parseFilesForWorkspace(wkspUri, testData, ctrl, 'configWatcher', false);
       } catch (e: unknown) {
         config.logger.showError(e, wkspUri);
       }
     }, DEBOUNCE_MS);

     configDebounceTimers.set(key, timer);
   };
   ```
4. Register all three event handlers (WATCH-02, PITFALL-10):
   ```typescript
   watcher.onDidCreate(uri => handler(uri, 'create'));
   watcher.onDidChange(uri => handler(uri, 'change'));
   watcher.onDidDelete(uri => handler(uri, 'delete'));
   ```
5. Return `[watcher]`.

**Key design notes:**
- Per-workspace debounce key is `wkspUri.path` (matches `fileParser.ts` pattern per PATTERNS.md).
- Timer Map is module-level, NOT class-level (configWatcher is a function module like workspaceWatcher, not a class).
- `getUrisOfWkspFoldersWithFeatures(true)` with forceRefresh=true clears the discoveryCache and rebuilds it.
- `onConfigChanged([wkspUri], true)` passes `clearNotifiedErrors=true` per WATCH-06 so fix-then-break cycles re-notify.
- `parser.parseFilesForWorkspace(...)` triggers `onStatusChange(busy=true)` internally, which sets status bar to "Behave: Parsing..." per D-01. No extra wiring needed.
- Do NOT route through `configurationChangedHandler` — it has an `integrationTestRun` early-exit guard at line 570 that would silently skip re-discovery during integration tests (PITFALL-04/14).
